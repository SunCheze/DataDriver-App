# DataDriver
Data Driver
Платформа для построения аналитической экосистемы и ускорения внедрения Data Driven-культуры в малом бизнесе (e-commerce).

🎯 Ключевая метрика продукта
Сокращение времени от запроса «хочу видеть метрики» до первых рабочих дашбордов и гипотез: с 1–3 месяцев до 3–7 дней.

🛠 Технологический стек (MVP)
Backend / ETL: Python, Apache Airflow (оркестрация), Pandas
База данных: PostgreSQL (основная), ClickHouse (для аналитических витрин - на будущее)
ML Ядро: Scikit-learn, LightGBM (предиктивная аналитика, расчет маржинальности)
Frontend (Веб-интерфейс): Streamlit (быстрый прототип), Django (переход на production-версию)
Инфраструктура: Docker, MLflow (версионирование моделей)

Логическая архитектура Data Driver
Концепция
Data Driver берет разрозненные данные e-commerce (сайт, маркетплейсы, Google Sheets) и превращает их в "единую версию правды" (чистую маржу по SKU) с помощью ML-моделей и готовых ETL-конвейеров.

Схема данных

Ключевые технические вызовы
Интеграции: Разные форматы API маркетплейсов, задержки в выдаче финансовых отчетов (возвраты).
Модель маржинальности: Построение динамической формулы расчета "чистой маржи", учитывающей меняющиеся комиссии МП и акции.

# Требования к MVP (на основе CustDev)

Фокус MVP: Владелец e-com (сайт + 1-3 МП), который хочет видеть реальную маржу по SKU без найма data-команды.

## Epic 1: Единая модель маржинальности
*   **US1.1:** Как владелец, я хочу подключить выгрузки/API из WB, Ozon и своего сайта, чтобы данные собирались в одном месте.
*   **US1.2:** Как владелец, я хочу видеть расчет "чистой маржи" по каждому товару (Выручка - Себестоимость - Комиссия МП - Логистика - Возвраты - Реклама), чтобы понимать реальную прибыльность SKU.
*   **US1.3:** Как система, я должна автоматически обновлять данные по марже не реже 1 раза в сутки (цель: "день в день").

## Epic 2: Рекомендательная система (ML)
*   **US2.1:** Как владелец, я хочу получать от системы рекомендации (например: "Товары X теряют маржу из-за роста логистики, повысьте цену на Y%"), чтобы принимать решения быстрее.
*   **US2.2:** Как владелец, я хочу иметь возможность запросить данные текстом (NLP-интерфейс: "Покажи товары с падающей конверсией"), чтобы не искать нужную кнопку в интерфейсе.

## Epic 3: Проверка гипотез
*   **US3.1:** Как владелец, я хочу создать простую A/B гипотезу (например, изменение цены) и отследить, как она повлияла на маржу в дашборде, без привлечения аналитиков.

## Non-Functional Requirements (НФТ)
*   **Time-to-value:** Первый дашборд на тестовых данных должен собираться за 10 минут.
*   **Security:** Строгая изоляция данных разных клиентов (multi-tenancy на уровне БД).
